# Stow-as-View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `stow(value, name, shape = "view", label = ...)` so a user can pre-stow per-window views from a dbplyr expression and consume them in a sweep via `mr_variant()`.

**Architecture:** Reuses `.mr_register_view()` (already in `R/launch_sql.R`) for the DDL + `_mr_versions` row. Adds a small input sniffer that scans the rendered SQL against the `_mr_versions` and `_mr_append_tables` registries (no dbplyr-internal walking, no `grab()`-time tagging — registry-driven matching is simpler and survives any verb chain). Writes a synthetic `_mr_runs` row with the supplied `variant_label` so `mr_variant()` resolution finds the view in a downstream sweep. A small companion change exposes `label =` on `stow()` so free-standing stows (outside a launch) can carry a variant label, which is currently only possible via launch propagation.

**Tech Stack:** R, DuckDB (via DBI), dbplyr, qs2 (already used). No new dependencies.

---

## Design notes (for the implementer)

Two key decisions made in the design conversation:

1. **`shape = "view"` over `materialize = FALSE`.** Joins the existing `shape =` enum at the call site. View identity is hashed by SQL text only (the `mr_sql()` precedent from `launch_sql.R:316`). Source-data staleness through views over append-shape inputs is a known weak spot; tracked in `TODO.md` under "Surfaced 2026-05-01."

2. **No new reference constructor.** The rolling-window sweep redirects via existing `mr_variant(label)`. Each fold gets its own variant label at stow-time; the same label resolves the rebind at launch-time. This requires `stow()` to accept `label =` so free-standing stows can attach a variant.

The vignette-target call site that motivated this work:

```r
purrr::pwalk(windows, function(fold, train_start_year, train_end_year, test_year) {
  fold_label <- sprintf("fold_%02d", fold)
  panel <- grab("cprice__pf")

  panel |> filter(year >= train_start_year, year <= train_end_year) |>
    stow("train", shape = "view", label = fold_label)
  panel |> filter(year == test_year) |>
    stow("test",  shape = "view", label = fold_label)
})

purrr::pwalk(windows, function(fold, ...) {
  fold_label <- sprintf("fold_%02d", fold)
  launch(model_code, rebind = list(
    train = mr_variant(fold_label),
    test  = mr_variant(fold_label)
  ), label = fold_label)
})
```

```r
## View-shape stow: dbplyr lazy expression -> CREATE OR REPLACE VIEW.
##
## Hashing follows the precedent set by .mr_register_view() in
## launch_sql.R: the rendered SQL text is the content_hash input. View
## identity therefore drifts when (a) the user changes the lazy
## expression or (b) an upstream input's physical name drifts (which
## happens for versioned-shape inputs but not for append-shape inputs).

# Scan a rendered SQL string for tokens that match physical names
# registered in `_mr_versions` or `_mr_append_tables`. Returns a list
# of (name, hash) pairs. Errors if no managed names are found ("strict"
# input resolution: a view-stow with no upstream provenance is almost
# certainly a user mistake).
#
# Detection is lexical -- tokenizes on word characters and matches
# against the registries. The two registries have disjoint physical-
# name conventions (`name__hex_hash` for versioned, `name__append` for
# append) so a token can only match one or the other. False positives
# would require a user to have a string literal in the SQL whose value
# coincides with a physical name; unlikely but acknowledged.
.mr_sniff_view_inputs <- function(con, rendered_sql) {
  tokens <- unique(regmatches(
    rendered_sql,
    gregexpr("[A-Za-z_][A-Za-z_0-9]*", rendered_sql)
  )[[1]])
  if (length(tokens) == 0L) {
    stop(
      "stow(shape = 'view'): no modelrunnR-managed inputs found in expression. ",
      "Lazy expressions for view-stow must derive from grab().",
      call. = FALSE
    )
  }

  # Build the IN-list once. DuckDB doesn't accept a vector-bound IN
  # via DBI parameterization in older versions, so render the list
  # inline using .mr_quote_str (existing helper) for safety.
  in_list <- paste(vapply(tokens, .mr_quote_str, character(1)), collapse = ", ")

  versioned <- DBI::dbGetQuery(con, sprintf(
    "SELECT physical_name, logical_name, content_hash
       FROM _mr_versions
      WHERE physical_name IN (%s)", in_list
  ))
  appended <- DBI::dbGetQuery(con, sprintf(
    "SELECT physical_name, logical_name
       FROM _mr_append_tables
      WHERE physical_name IN (%s)", in_list
  ))

  if (nrow(versioned) + nrow(appended) == 0L) {
    stop(
      "stow(shape = 'view'): no modelrunnR-managed inputs found in expression. ",
      "Lazy expressions for view-stow must derive from grab().",
      call. = FALSE
    )
  }

  inputs <- vector("list", nrow(versioned) + nrow(appended))
  i <- 1L
  for (j in seq_len(nrow(versioned))) {
    inputs[[i]] <- list(
      name = versioned$logical_name[j],
      hash = versioned$content_hash[j]
    )
    i <- i + 1L
  }
  for (j in seq_len(nrow(appended))) {
    inputs[[i]] <- list(
      name = appended$logical_name[j],
      hash = NA_character_
    )
    i <- i + 1L
  }
  inputs
}
```

If `.mr_quote_str` doesn't exist, use `paste0("'", gsub("'", "''", tokens, fixed = TRUE), "'")` inline instead.

```r
# Orchestrator for shape = "view" stows. Renders the lazy expression
# to SQL, sniffs managed inputs, registers the view via the existing
# `.mr_register_view()` helper, and writes a synthetic _mr_runs row so
# `mr_variant(label)` resolves to this view in a downstream sweep.
.mr_stow_view <- function(name, value, label = NULL) {
  con <- .mr_get_connection()

  rendered <- as.character(dbplyr::sql_render(value, con = con))
  if (length(rendered) != 1L || !nzchar(rendered)) {
    stop("stow(shape = 'view'): could not render the lazy expression to SQL.",
         call. = FALSE)
  }

  # Strict input resolution.
  inputs <- .mr_sniff_view_inputs(con, rendered)

  # Namespace guard before any DDL.
  .mr_guard_namespace(name, shape = "A", new_kind = "view", context = "stow")

  # Register the view. Returns the SQL-text content_hash and writes the
  # `_mr_versions` row + CREATE OR REPLACE VIEW. The DDL fires inside
  # .mr_register_view's own transaction.
  hash <- .mr_register_view(name, rendered)

  # Write the synthetic run row outside .mr_register_view's transaction.
  # This row is what `mr_variant(label)` walks to find the view.
  if (!.mr_is_recording() && !isTRUE(.mr_state$suppress_interactive)) {
    .mr_write_view_interactive_run_row(con, name, hash, inputs, label)
  }

  invisible(hash)
}

# Shape A "interactive" run row for a view-stow. Distinct from
# `.mr_maybe_record_interactive_write` because it records the sniffed
# inputs (versioned and append) on the row, not just the output pair.
.mr_write_view_interactive_run_row <- function(con, name, hash, inputs, label) {
  now <- Sys.time()
  step <- sprintf("<interactive:%s>", format(now, "%Y-%m-%d %H:%M:%OS3"))
  si <- .mr_capture_session_info()

  inputs_json  <- .mr_pairs_to_json(inputs)
  outputs_json <- .mr_pairs_to_json(list(.mr_pair(name, hash)))

  row <- data.frame(
    step              = step,
    run_id            = .mr_new_run_id(),
    inputs            = inputs_json,
    outputs           = outputs_json,
    started_at        = now,
    duration_ms       = 0L,
    status            = "interactive",
    variant_label     = if (is.null(label)) NA_character_ else label,
    hostname          = si$hostname,
    os                = si$os,
    arch              = si$arch,
    r_version         = si$r_version,
    n_cpu             = si$n_cpu,
    total_ram_bytes   = si$total_ram_bytes,
    free_ram_bytes    = si$free_ram_bytes,
    attached_packages = si$attached_packages,
    git_sha           = si$git_sha,
    git_branch        = si$git_branch,
    git_dirty         = si$git_dirty,
    stringsAsFactors  = FALSE
  )
  DBI::dbAppendTable(con, "_mr_runs", row)
  invisible(NULL)
}
```

```r
test_that("rolling-window pattern: per-fold views + mr_variant rebind", {
  new_test_db()
  con <- .mr_get_connection()

  # Source panel: 11 years.
  ingest(data.frame(year = 2014:2024, x = seq_len(11)), "panel",
         shape = "versioned")

  # Two folds: train on 2014-2020/test 2021, train on 2015-2021/test 2022.
  windows <- data.frame(
    fold             = 1:2,
    train_start_year = c(2014L, 2015L),
    train_end_year   = c(2020L, 2021L),
    test_year        = c(2021L, 2022L)
  )

  # Pre-stow per-fold views.
  for (i in seq_len(nrow(windows))) {
    fold_label <- sprintf("fold_%02d", windows$fold[i])
    panel <- grab("panel")
    panel |>
      dplyr::filter(year >= windows$train_start_year[i],
                    year <= windows$train_end_year[i]) |>
      stow("train", shape = "view", label = fold_label)
    panel |>
      dplyr::filter(year == windows$test_year[i]) |>
      stow("test", shape = "view", label = fold_label)
  }

  # The two folds produced two variants of `train` and `test`.
  variants <- DBI::dbGetQuery(con,
    "SELECT DISTINCT variant_label FROM _mr_runs
      WHERE variant_label LIKE 'fold_%' ORDER BY variant_label")
  expect_identical(variants$variant_label, c("fold_01", "fold_02"))

  # Each variant resolves to a distinct view hash.
  for (i in seq_len(nrow(windows))) {
    fold_label <- sprintf("fold_%02d", windows$fold[i])
    h_train <- .mr_latest_hash_for_variant(con, "train", fold_label)
    h_test  <- .mr_latest_hash_for_variant(con, "test",  fold_label)
    expect_false(is.null(h_train))
    expect_false(is.null(h_test))
  }

  # And the launch-side rebind path resolves cleanly:
  out <- launch(quote({
    df <- grab("train") |> dplyr::collect()
    stow(data.frame(n_rows = nrow(df)), "fold_metric", shape = "append")
  }), rebind = list(train = mr_variant("fold_01")), label = "fold_01")
  expect_identical(out$status, "success")

  metric_rows <- DBI::dbGetQuery(con,
    "SELECT n_rows, _mr_variant_label FROM fold_metric__append
      ORDER BY _mr_variant_label")
  expect_identical(metric_rows$n_rows[1], 7L)  # 2014..2020
  expect_identical(metric_rows[["_mr_variant_label"]][1], "fold_01")
})
```

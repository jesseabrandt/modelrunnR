# Lazy `grab()` and server-side `stow()` — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `grab()` return a `dbplyr` lazy tbl for tabular values, let `stow()` accept a lazy tbl and realize it server-side, turn `ingest()` into a no-R-memory CSV loader, expose a connection-getter `mr_con()`, and add a `duckdb_seed` arg to `launch()` for reproducible in-database sampling. Ships an end-to-end lazy path so large CSVs never round-trip through R.

**Architecture:** Three helper layers plumb the behavior. (1) A factored `.mr_hash_duckdb_table(con, name)` replaces the inner body of `.mr_hash_frame()` so ingest, materialized stow, and lazy stow all share one hashing implementation. (2) `ingest()` is rewritten around `CREATE TABLE <physical> AS SELECT * FROM read_csv_auto(...)` and never loads the frame into R. (3) `stow()` adds a `tbl_lazy` dispatch branch that `CREATE TABLE AS`-es the rendered SQL. `grab()`'s table-kind branch switches from `dbReadTable()` to `dplyr::tbl(con, physical_name)`. The schema gains two nullable columns (`_mr_versions.source_sql`, `_mr_runs.duckdb_seed`) for durable provenance; no existing columns change.

**Tech Stack:** R 4.5, DuckDB via `duckdb` + `DBI`, `dplyr` + `dbplyr` for lazy tbl construction and SQL rendering, `testthat` 3e, `roxygen2`, `devtools`.

**Spec:** `docs/superpowers/specs/2026-04-17-lazy-grab-design.md`

```r
# Content-hash an existing DuckDB table by name. Row- and column-order
# independent, type-sensitive. See .mr_hash_frame for the algorithm
# rationale and caveats (collision bound, type sensitivity).
.mr_hash_duckdb_table <- function(con, table_name) {
  info <- DBI::dbGetQuery(
    con,
    sprintf("PRAGMA table_info(%s)", .mr_quote_ident(table_name))
  )
  cols <- sort(info$name)

  if (length(cols) == 0L) {
    # Table with no columns — shouldn't happen in practice; hash on empty key.
    empty_key <- "empty|"
    sql <- sprintf("SELECT MD5('%s') AS h", empty_key)
    return(DBI::dbGetQuery(con, sql)$h[[1]])
  }

  nrows <- DBI::dbGetQuery(
    con,
    sprintf("SELECT COUNT(*) AS n FROM %s", .mr_quote_ident(table_name))
  )$n[[1]]

  if (nrows == 0L) {
    empty_key <- paste0("empty|", paste(cols, collapse = ","))
    sql <- sprintf("SELECT MD5('%s') AS h", gsub("'", "''", empty_key, fixed = TRUE))
    return(DBI::dbGetQuery(con, sql)$h[[1]])
  }

  cols_sql <- paste(vapply(cols, .mr_quote_ident, character(1)), collapse = ", ")
  sql <- sprintf(
    "SELECT MD5(STRING_AGG(CAST(HASH(%s) AS VARCHAR), '|' ORDER BY HASH(%s))) AS h FROM %s",
    cols_sql, cols_sql, .mr_quote_ident(table_name)
  )
  DBI::dbGetQuery(con, sql)$h[[1]]
}

# Content-hash a data frame by writing it to a transient DuckDB temp
# table and hashing that table. Kept as a thin wrapper so the existing
# materialized-stow path is unchanged.
.mr_hash_frame <- function(con, df) {
  if (nrow(df) == 0L) {
    # Short-circuit for empty frames: no need to round-trip through
    # DuckDB; replicate the same "empty|<sorted,colnames>" key.
    cols <- sort(names(df))
    empty_key <- paste0("empty|", paste(cols, collapse = ","))
    sql <- sprintf("SELECT MD5('%s') AS h", gsub("'", "''", empty_key, fixed = TRUE))
    return(DBI::dbGetQuery(con, sql)$h[[1]])
  }

  tmp <- paste0(
    "_mr_tmp_hash_",
    paste(sample(c(0:9, letters), 10, replace = TRUE), collapse = "")
  )
  .mr_table_write(con, tmp, df, overwrite = TRUE)
  on.exit(try(.mr_drop_table(con, tmp), silent = TRUE), add = TRUE)
  .mr_hash_duckdb_table(con, tmp)
}
```

```r
# Server-side CSV/Parquet ingest: CREATE TABLE AS from DuckDB's
# read_csv_auto / read_parquet. No R-side materialization.
#
# Returns the name of the newly-created DuckDB table. The caller is
# responsible for hashing, renaming to the canonical physical_name,
# registering in _mr_versions, and cleaning up on error.
.mr_ingest_file_to_table <- function(con, path, dest_table) {
  if (any(charToRaw(path) == as.raw(0L))) {
    stop("ingest(): path contains a null byte.", call. = FALSE)
  }
  normalized <- normalizePath(path, mustWork = FALSE)
  if (!file.exists(normalized)) {
    stop(sprintf("ingest(): file does not exist: %s", path), call. = FALSE)
  }
  ext <- tolower(tools::file_ext(normalized))
  reader <- switch(
    ext,
    csv     = "read_csv_auto",
    tsv     = "read_csv_auto",
    parquet = "read_parquet",
    pq      = "read_parquet",
    stop(sprintf("ingest(): unsupported file extension '%s'", ext), call. = FALSE)
  )
  escaped <- gsub("'", "''", normalized, fixed = TRUE)
  sql <- sprintf(
    "CREATE TABLE %s AS SELECT * FROM %s('%s')",
    .mr_quote_ident(dest_table), reader, escaped
  )
  .mr_execute(con, sql)
  invisible(dest_table)
}
```

Replace `ingest()` in `R/ingest.R` (lines 23-50) with:

```r
#' @return A `dbplyr` lazy `tbl` over the ingested table, invisibly.
#'   Users typically don't use the return value directly — they call
#'   [grab()] to read the stored data by name.
#' @export
ingest <- function(name, source) {
  .mr_validate_name(name, context = "ingest")
  stopifnot(
    is.character(source),
    length(source) == 1L,
    nzchar(source)
  )
  if (!file.exists(source)) {
    stop(sprintf("ingest(): file not found: %s", source), call. = FALSE)
  }

  con <- .mr_get_connection()

  # Stage into a temp table name, hash it, then rename to the canonical
  # physical_name(name, hash) if this is a new version (or drop it if
  # the content already exists under this logical_name).
  staging <- paste0(
    "_mr_tmp_ingest_",
    paste(sample(c(0:9, letters), 10, replace = TRUE), collapse = "")
  )
  .mr_ingest_file_to_table(con, source, staging)
  on.exit(try(.mr_drop_table(con, staging), silent = TRUE), add = TRUE)

  content_hash <- .mr_hash_duckdb_table(con, staging)
  physical_name <- .mr_physical_name(name, content_hash)
  src_hash      <- .mr_file_hash(source)
  src_uri       <- normalizePath(source, mustWork = TRUE)

  existing <- .mr_get_version_row(con, name, content_hash)
  now <- Sys.time()

  DBI::dbBegin(con)
  tryCatch({
    if (nrow(existing) == 0L) {
      # Promote staging to the canonical physical_name.
      .mr_execute(
        con,
        sprintf(
          "ALTER TABLE %s RENAME TO %s",
          .mr_quote_ident(staging), .mr_quote_ident(physical_name)
        )
      )
      # size_bytes: DuckDB doesn't expose per-table bytes cheaply; use
      # COUNT(*) * narrow-approximation is worse than just storing 0 and
      # revisiting. Record NULL/0 for now; this column is advisory.
      size_bytes <- 0
      DBI::dbExecute(
        con,
        "INSERT INTO _mr_versions
           (logical_name, content_hash, physical_name, kind,
            first_seen, last_seen, size_bytes, storage_location,
            source_uri, source_hash)
         VALUES (?, ?, ?, 'table', ?, ?, ?, NULL, ?, ?)",
        params = list(name, content_hash, physical_name, now, now,
                      size_bytes, src_uri, src_hash)
      )
      .mr_refresh_latest_view(con, name)
    } else {
      DBI::dbExecute(
        con,
        "UPDATE _mr_versions
           SET last_seen = ?, source_uri = ?, source_hash = ?
         WHERE logical_name = ? AND content_hash = ?",
        params = list(now, src_uri, src_hash, name, content_hash)
      )
    }
    DBI::dbCommit(con)
  }, error = function(e) {
    DBI::dbRollback(con)
    stop(e)
  })

  .mr_record_write(name, content_hash)
  .mr_maybe_record_interactive_write(name, content_hash)
  .mr_maybe_warn_version_count(con, name)

  invisible(dplyr::tbl(con, physical_name))
}
```

Create `R/stow_lazy.R`:

```r
# Server-side stow of a dbplyr lazy tbl.
#
# Flow:
#   1. Verify the tbl is bound to the modelrunnR connection.
#   2. Render the SQL.
#   3. CREATE TABLE <staging> AS <sql>, hash it.
#   4. If this content_hash already exists under this logical_name:
#      drop the staging table, bump last_seen.
#   5. Else: rename staging to canonical physical_name, insert the
#      _mr_versions row with source_sql populated, refresh the view.
#   6. Record the write on the current run row (same path as .mr_stow_table).
.mr_stow_lazy <- function(name, value) {
  con <- .mr_get_connection()
  remote <- dbplyr::remote_con(value)
  if (!identical(remote, con)) {
    stop(
      "stow(): lazy tbl is bound to a different DBI connection; ",
      "call dplyr::collect() first and stow the materialized result.",
      call. = FALSE
    )
  }

  sql_text <- as.character(dbplyr::sql_render(value))

  staging <- paste0(
    "_mr_tmp_stow_",
    paste(sample(c(0:9, letters), 10, replace = TRUE), collapse = "")
  )
  .mr_execute(
    con,
    sprintf(
      "CREATE TABLE %s AS %s",
      .mr_quote_ident(staging), sql_text
    )
  )
  # Track staging so any error below drops it.
  staging_alive <- TRUE
  on.exit(
    if (staging_alive) try(.mr_drop_table(con, staging), silent = TRUE),
    add = TRUE
  )

  content_hash <- .mr_hash_duckdb_table(con, staging)
  physical_name <- .mr_physical_name(name, content_hash)

  existing <- .mr_get_version_row(con, name, content_hash)
  now <- Sys.time()

  DBI::dbBegin(con)
  tryCatch({
    if (nrow(existing) == 0L) {
      .mr_execute(
        con,
        sprintf(
          "ALTER TABLE %s RENAME TO %s",
          .mr_quote_ident(staging), .mr_quote_ident(physical_name)
        )
      )
      staging_alive <- FALSE  # renamed away; no drop on exit
      DBI::dbExecute(
        con,
        "INSERT INTO _mr_versions
           (logical_name, content_hash, physical_name, kind,
            first_seen, last_seen, size_bytes, storage_location,
            source_sql)
         VALUES (?, ?, ?, 'table', ?, ?, 0, NULL, ?)",
        params = list(name, content_hash, physical_name, now, now, sql_text)
      )
      .mr_refresh_latest_view(con, name)
    } else {
      DBI::dbExecute(
        con,
        "UPDATE _mr_versions
           SET last_seen = ?
         WHERE logical_name = ? AND content_hash = ?",
        params = list(now, name, content_hash)
      )
      # staging will be dropped on exit
    }
    DBI::dbCommit(con)
  }, error = function(e) {
    DBI::dbRollback(con)
    stop(e)
  })

  .mr_record_write(name, content_hash)
  .mr_maybe_record_interactive_write(name, content_hash)
  .mr_maybe_warn_version_count(con, name)
  invisible(content_hash)
}
```

````markdown
modelrunnR stores tables in DuckDB. When you `grab()` a tabular value, you

write.csv(
  data.frame(
    region = sample(c("N", "S", "E", "W"), 50000, replace = TRUE),
    year   = sample(2018:2024, 50000, replace = TRUE),
    price  = rlnorm(50000, meanlog = 11, sdlog = 0.5),
    sqft   = round(rnorm(50000, mean = 1800, sd = 400))
  ),
  tmp, row.names = FALSE
)

raw <- grab("sales", source = tmp)
class(raw)
```

`grab()` ingested the CSV server-side via DuckDB's `read_csv_auto()` —
no R-side materialization. Subsequent `grab("sales")` calls return a
lazy tbl directly.

## Composing `dplyr` verbs stays lazy

```{r verbs}
raw |>
  filter(year >= 2020) |>
  group_by(region) |>
  summarise(n = n(), avg_price = mean(price), .groups = "drop")
```

That ran as SQL inside DuckDB. Use `show_query()` to see exactly what
DuckDB executed:

```{r show-query}
raw |>
  filter(year >= 2020) |>
  group_by(region) |>

```

## Materializing

Three idioms — pick whichever reads most naturally:

```{r collect}
# dplyr::collect()
sample1 <- raw |> slice_sample(n = 100) |> collect()

# as.data.frame()
sample2 <- as.data.frame(raw |> slice_sample(n = 100))

# tibble::as_tibble()
sample3 <- tibble::as_tibble(raw |> slice_sample(n = 100))
```

Non-`dplyr` consumers that coerce via `as.data.frame()` (most of base
`stats`, `ggplot2`) auto-collect transparently:

```{r stats}
fit <- lm(price ~ sqft, data = raw |> slice_sample(n = 1000) |> collect())
coef(fit)
```

Two things *don't* auto-collect: `x$col` (base `$` isn't defined on
`tbl_lazy`) and base `[` subsetting. Use `dplyr::pull(col)` or collect
first.

## Reproducible sampling with `duckdb_seed`

`dplyr::slice_sample()` against a lazy tbl uses DuckDB's RNG — R's
`set.seed()` does not reach it. Pass `duckdb_seed` to `launch()`:

```{r seed-run}
launch(
  {
    sample <- grab("sales") |>
      slice_sample(n = 1000) |>
      collect()
    stow(sample, "sales_sample")
  },
  label   = "sample_demo",
  duckdb_seed = 0.42
)
```

Every `launch()` under the same `duckdb_seed` produces a byte-identical
sample. Valid range is `[-1, 1]`; the value is recorded on the run
row for reproducibility audits.

## Whole-step server-side pipelines

`stow()` accepts a lazy tbl. The entire pipeline step runs as SQL — no
R-side materialization, even at the end:

```{r lazy-stow}
launch(
  {
    grab("sales") |>
      filter(year == 2024) |>
      group_by(region) |>
      summarise(total = sum(price), .groups = "drop") |>
      stow("sales_by_region_2024")
  },
  label = "regional_rollup"
)

# Downstream: grab() returns another lazy tbl — composes right back in.
grab("sales_by_region_2024") |> collect()
```

The rendered SQL is stored alongside the output on `_mr_versions.source_sql`
for later inspection.

## Escape hatches

For sampling schemes `dbplyr` doesn't express — stratified, grouped CV,
custom — two reliable exits:

**Raw SQL via `mr_con()`.** The logical name (`sales`) is a view over
the latest version, so you can query it directly:

```{r raw-sql}
DBI::dbGetQuery(
  mr_con(),
  "SELECT region, COUNT(*) AS n
   FROM sales
   USING SAMPLE 5 PERCENT (reservoir, 42)
   GROUP BY region"
)
```

That's DuckDB's stratified-by-region reservoir sample, seeded inline —
one way to express sampling schemes `dbplyr` doesn't cover.

**Collect then sample in R.** When the sampling logic is cleaner in R
— grouped CV, rolling windows, stratification — pull the keys you
need, sample in R, stow the partition:

```{r r-side-index}
# Pull just the region + year columns (cheap); build folds in R.
keys <- grab("sales") |>
  select(region, year) |>
  mutate(rid = row_number()) |>
  collect()
# ... R-side sampling / fold construction over `keys` ...
```

Use whichever fits the structure you're building.
````
